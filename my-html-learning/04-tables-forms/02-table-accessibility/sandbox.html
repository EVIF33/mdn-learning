Table Accessibility for Screen Readers

Main rule: Every table must work with screen readers

Screen readers announce tables differently. Without proper markup, users hear: "Table with 3 rows, 4 columns" instead of meaningful data.

1. Table Caption: Context Matters

❌ Without caption:

Month Sales
January $5,000
Screen reader says:
"Table with 2 rows and 2 columns"
"Column 1: Month"
"Column 2: Sales"
"Row 1: January, $5,000"

✅ With caption:

Monthly Sales Report - North Region
Month Sales
January $5,000
Screen reader says:
"Table: Monthly Sales Report - North Region"
"Column 1: Month"
"Column 2: Sales"
"Row 1: January, $5,000" 
            <table>
    <caption>Monthly Sales Report - North Region</caption>
    <tr>
        <th>Month</th>
        <th>Sales</th>
    </tr>
</table>

2. Scope Attribute: Row vs Column Headers

Product Inventory
Product Category Price Stock
MacBook Pro Laptops $1,299 15
iPhone 15 Phones $799 42
Screen reader navigation:
"Table: Product Inventory"
"Column 1: Product"
"Column 2: Category"
"Row 1: MacBook Pro"
"Category: Laptops"
"Price: $1,299"
"Stock: 15" 
            <tr>
    <th scope="col">Product</th>  <!-- Column header -->
    <th scope="col">Category</th>
</tr>
<tr>
    <th scope="row">MacBook Pro</th>  <!-- Row header -->
    <td>Laptops</td>
</tr>

3. Complex Tables: id and headers Attributes

For tables with multiple header levels or merged cells:

Project Timeline 2024
Project Q1 Q2
Plan Execute Plan Execute
Web App Design Frontend Testing Launch
Screen reader says for "Testing" cell:
"Web App, Q2, Plan: Testing" 
            <!-- Header with id -->
<th id="web-app" headers="project">Web App</th>
            
<!-- Data cell linked to headers -->
<td headers="web-app q2 q2-plan">Testing</td>

4. ❌ Common Accessibility Failures

Bad: Missing scope

Name Email
Alex alex@test.com
Screen reader confusion:
"Name" - Is this a column or row header?
Reads cells without proper context

Bad: Layout table without role

Sidebar

  • Menu 1
  • Menu 2

Main Content

Content here...
Screen reader says:
"Table with 1 row and 2 columns"
"Cell: Sidebar, Menu 1, Menu 2"
"Cell: Main Content, Content here..."
⚠️ Confusing! This should be a layout, not a data table 
            <!-- Correct: Layout table -->
<table role="presentation">
    <!-- Screen reader ignores this table -->
</table>

5. Semantic Structure: thead, tbody, tfoot

Annual Budget Summary
Department Q1 Q2 Total
Marketing $15,000 $20,000 $35,000
Development $50,000 $60,000 $110,000
Grand Total $65,000 $80,000 $145,000
Screen reader navigation:
"Table: Annual Budget Summary"
"Header section:"
"Column 1: Department"
"Body section:"
"Row 1: Marketing"
"Footer section:"
"Row: Grand Total" 
            <table>
    <caption>...</caption>
                
    <thead>    <!-- Header section -->
        <tr>...</tr>
    </thead>
                
    <tbody>    <!-- Main data -->
        <tr>...</tr>
    </tbody>
                
    <tfoot>    <!-- Summary -->
        <tr>...</tr>
    </tfoot>
</table>
